## Usage Rules

- Add only terms that need consistency or project-specific decisions.
- Include notes for gender, register, UI length, lore constraints, or accepted variants.
- Do not change existing glossary terms during QA unless they are clearly wrong or the user asks for glossary cleanup.
- When a source term has several Bulgarian options, record the selected one and explain when variants are allowed.
